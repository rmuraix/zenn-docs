## 背景

動画データを用いた研究、例えばTemporal Action Localizationなどでは、動画内のどの時間区間でどのイベントが発生しているかをラベルとして付与する作業が必要になります。

このようなアノテーションを行うためのツールとして、例えば [VGG Image Annotator (VIA)](https://www.robots.ox.ac.uk/~vgg/software/via/) や [Label Studio](https://labelstud.io/) などの汎用的なアノテーションツールが存在します。
しかし、これらのツールは動画編集ソフトウェアとは独立しているため、動画素材の編集とアノテーション作業が分断されてしまいます。
また、動画編集に慣れているユーザーにとっては、編集ソフトとは異なる操作体系のツールを使用すること自体が手間になる場合もあります。

このような背景から、動画編集環境の中で直接アノテーションを行えるツールがあれば、作業効率を向上させることができるのではと考えました。そこで、Adobe Premiere Pro上で動作する簡易的な拡張機能として、時系列アノテーションを行うツールを開発しました。

リポジトリはこちらです。

https://github.com/rmuraix/premianno

## 作った拡張機能

今回作った拡張機能「[Premianno](https://lab.rmurai.com/premianno/)」では、次のような手順を踏むことで、Premiere Pro上で動画に対して時系列アノテーションを行うことができます。

1. Premiere Pro上で動画を開き、シーケンスに対してアクションの範囲ごとにカットを入れる
2. 拡張機能のパネルを開き、各カットに対して、対応するアクションのクラスを選択する
3. アノテーションが完了したら、拡張機能からアノテーションデータをエクスポートする

![スクリーンショット](https://github.com/rmuraix/premianno/raw/main/packages/docs/images/overview.png)

クラスの選択は、あらかじめ定義されたクラスセットから選ぶことができます。例えば、以下のようなクラスセットをインポートして使用できます。

```csv
index,class
0,Walking
1,Running
2,OpeningDoor
```

### 出力フォーマット

エクスポートされたアノテーションデータはTOML形式で保存され、各カットの開始時間、終了時間、クラスラベルが含まれます。

```toml
[sequence]
id = "4f2e8..."
name = "Video Sequence 01"
timebase_ticks = "254016000000"
frame_rate = 29.97
project_path = "/path/to/project.prproj"
exported_at = "2026-02-19T00:00:00.000Z"

[[intervals]]
start_seconds = 0
end_seconds = 3.336667
duration_frames = 100
order_index = 0
label = "Walking"
```

このように、Premiere Proのタイムラインでカットを作成し、それぞれにラベルを付与することで、動画の時系列アノテーションを効率的に作成できます。

### インストール方法

1. [リリースページ](https://github.com/rmuraix/premianno/releases)から最新の`*zxp`ファイルをダウンロードする
2. [ZXP/UXP Installer](https://aescripts.com/learn/zxp-installer/)をインストールして起動する
3. ダウンロードした`*zxp`ファイルをインストーラーにドラッグ&ドロップしてインストールする

https://lab.rmurai.com/premianno/ja/guide/getting-started.html

## CEP拡張とは

CEP (Common Extensibility Platform) は、Adobe Creative Cloudアプリに対して、JavaScript等を用いて機能を追加できるものです。
CEP拡張は、アプリケーション内にパネルとして表示されるUIを提供でき、JavaScriptからAdobeのスクリプトAPIを呼び出すことで、アプリケーションの機能と連携できます。

近年、AdobeはUXP (Unified Extensibility Platform) という新しい拡張フレームワークも提供しています。しかしPremiere Proにおいては、UXPのサポートは比較的新しく、[バージョン25.6以降でのみ利用可能](https://developer.adobe.com/premiere-pro/uxp/introduction/#support-and-features)となっています。
そのため、より多くの環境で動作することを考え、本ツールではCEP拡張として実装する方法を採用しました。
ただし、将来的にCEPは非推奨化されUXPに完全移行する可能性があるため、今後のAdobeの動向には注意が必要です。

開発には、bolt-cepというフレームワークを利用しています。

https://github.com/hyperbrew/bolt-cep

bolt-cepはReact / Vue / SvelteやTypeScriptを用いてCEP拡張を開発するためのツールで、Adobe APIとの通信や開発環境の構築を簡単に行うことができます。
bolt-cepの基本的な使い方については、以下の記事が参考になります。

https://qiita.com/yuki_works/items/2372174a579f6f519aa9

## 実装のポイント

### アーキテクチャ概要

Premiannoは **CEPパネル(UI)** と **ExtendScript(Premiere操作)** を分離する構成です。

```
React/TypeScript UI (CEP Panel)
  ├─ 状態管理・UI
  ├─ 保存/読込 (userData)
  └─ TOML 生成
        ↓ evalTS
ExtendScript (jsx)
  ├─ Premiere API 呼び出し
  └─ カット/シーケンス情報取得
        ↓
Premiere Pro
```

### カット区間の抽出

Premiere Proのタイムラインでは、動画は **トラック上のクリップとして配置**されています。
そのため、動画全体をアノテーション区間として扱うには、**クリップの境界を集めて区間を構築する処理**が必要になります。

Premiannoでは、次の手順で区間を生成しています。

1. シーケンスのクリップ境界を取得する
2. 境界の重複を除去する
3. 時刻順にソートする
4. 隣接する境界を結んで区間を作る

この処理によって、編集済みのカット構造に基づいた区間を自動的に生成できます。

```ts
const boundaries = getTrackBoundaries(sequence);
boundaries.push(0);
if (endSeconds > 0) boundaries.push(endSeconds);

const sorted = uniqueSorted(boundaries);
for (let i = 0; i < sorted.length - 1; i++) {
  const start = sorted[i];
  const end = sorted[i + 1];
  if (end <= start) continue;
  intervals.push({ startSeconds: start, endSeconds: end });
}
```

このように、タイムライン上の編集情報を利用することで、ユーザーはカットを入れるだけでアノテーション区間を定義できるようになります。

### ExtendScript とフロントエンドの通信

CEP拡張では、UI側のJavaScriptから直接Premiere ProのAPIを呼び出すことはできません。
その代わり、**ExtendScript** と呼ばれるAdobe独自のスクリプト環境を経由して操作します。

そのため、CEP拡張では次のような構成になります。

```
Frontend (React / TypeScript)
        ↓
evalTS
        ↓
ExtendScript
        ↓
Premiere Pro API
```

`bolt-cep` では、この通信を簡単にするため `evalTS` という関数が提供されています。

例えば、現在アクティブなシーケンスの情報を取得する処理は次のように書くことができます。

```ts
// CEP -> ExtendScript
export const getActiveSequenceInfo = async () => {
  return evalTS("getActiveSequenceInfo");
};
```

また、`listenTS` / `dispatchTS` を利用することで、**イベントベースの双方向通信**も可能になります。

### クラスリストの保持方法

読み込んだクラスリストはCEP側で `userData` 配下にJSONとして保存しています。以下のように実現できます。

```ts
const STORAGE_DIR = "premianno-annotations";
const CLASS_LIST_FILE = "class-list.json";

const getStorageDir = () => {
  const base = csi.getSystemPath("userData");
  const dir = path.join(base, STORAGE_DIR);
  if (!fs.existsSync(dir)) fs.mkdirSync(dir, { recursive: true });
  return dir;
};

export const loadClassList = (): string[] => {
  const filePath = path.join(getStorageDir(), CLASS_LIST_FILE);
  if (!fs.existsSync(filePath)) return [];
  return JSON.parse(fs.readFileSync(filePath, "utf8"));
};

export const saveClassList = (classes: string[]) => {
  const filePath = path.join(getStorageDir(), CLASS_LIST_FILE);
  fs.writeFileSync(filePath, JSON.stringify(classes, null, 2), "utf8");
};
```

## ハマったポイント

開発時、CEPのJSX初期化が完了する前にUI側から `scanCutIntervals` を呼んでしまい、TypeErrorが発生していました。
CEP拡張では、UI側（CEPパネル）とExtendScript（JSX）は別の環境で動作しており、UIがロードされた時点ではJSX側のスクリプトがまだ読み込まれていない場合があります。この状態で `evalTS` を呼び出すと、JSX側の関数が定義されておらずエラーになります。

対応として、JSXのロード完了を待ってからUIを起動するようにし、初期化の順序を明示的に保証しています。

```ts
// initBolt を async 化して JSX の eval 完了を待つ
export const initBolt = async (log = true) => {
  if (window.cep) {
    const extRoot = csi.getSystemPath("extension");
    const jsxSrc = `${extRoot}/jsx/index.js`;
    if (fs.existsSync(jsxSrc)) {
      if (log) console.log(jsxSrc);
      await evalFile(jsxSrc);
    }
    initializeCEP();
  }
};

// React の起動前に initBolt を await
const bootstrap = async () => {
  await initBolt();
  ReactDOM.createRoot(document.getElementById("app") as HTMLElement).render(
    <React.StrictMode>
      <App />
    </React.StrictMode>
  );
};
```

この修正で、初期化完了前の `evalTS` 呼び出しによるエラーを再現しなくなりました。

## まとめ

Premiere ProのCEP拡張として、動画の時系列アノテーションを行うツールを実装しました。

- Premiereのカット情報を利用してアノテーション区間を生成
- CEP / ExtendScriptを使ったPremiere API連携
- React + TypeScriptでUIを実装

動画編集環境の中でアノテーションを行えるようにすることで、
動画データセット作成のワークフローをシンプルにできる可能性があります。
